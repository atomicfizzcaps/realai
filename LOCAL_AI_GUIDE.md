# Your Own AI - Local Model Guide

RealAI now supports **running your own AI models locally** without any external API dependencies! This makes RealAI truly independent, private, and cost-free.

## 🎯 Why Use Local Models?

- **🔒 100% Private**: Your data never leaves your machine
- **💰 Zero API Costs**: No per-token charges, only infrastructure costs
- **🚀 No Rate Limits**: Process as much as your hardware allows
- **🌐 Offline Capable**: Works without internet connection
- **🎨 Full Control**: Customize, fine-tune, and own your AI
- **⚡ Fast Inference**: Direct hardware access, no network latency

## 📋 Quick Start

### 1. Install Local Model Dependencies

```bash
# For LLM inference with GGUF models (recommended - works on CPU and GPU)
pip install llama-cpp-python

# For Hugging Face models (requires more memory)
pip install transformers torch

# Optional: For image generation
pip install diffusers accelerate

# Optional: For local speech-to-text
pip install openai-whisper
```

### 2. Download a Model

The easiest way is to download GGUF models from Hugging Face:

```bash
# Install Hugging Face CLI
pip install huggingface-hub

# Download Llama 3.2 1B (fast, low memory - great for getting started!)
huggingface-cli download bartowski/Llama-3.2-1B-Instruct-GGUF \
  --include '*Q4_K_M.gguf' \
  --local-dir ./models/llama3.2-1b

# Or download Llama 3.2 3B (better quality)
huggingface-cli download bartowski/Llama-3.2-3B-Instruct-GGUF \
  --include '*Q4_K_M.gguf' \
  --local-dir ./models/llama3.2-3b

# Or download Phi-3 Mini (Microsoft, 3.8B parameters)
huggingface-cli download microsoft/Phi-3-mini-4k-instruct-gguf \
  --include '*q4.gguf' \
  --local-dir ./models/phi3-mini
```

### 3. Register Your Model

```bash
# Register the downloaded model
python local_model_cli.py register llama3.2-1b \
  ./models/llama3.2-1b/Llama-3.2-1B-Instruct-Q4_K_M.gguf

# This will automatically set it as your default model
```

### 4. Use It!

```python
from realai import RealAIClient

# Initialize with local mode
client = RealAIClient(provider="local")

# Or explicitly enable local models
client = RealAIClient(use_local=True)

# Chat like normal - but now it's YOUR AI!
response = client.chat.create(
    messages=[
        {"role": "user", "content": "Explain quantum computing in simple terms"}
    ]
)

print(response['choices'][0]['message']['content'])
# Response generated by YOUR local AI model!
```


- Quality: Very good for most tasks
- Speed: Fast on GPU, acceptable on CPU

## 🔄 Hybrid Mode

You can use both local and API models together:

```python
from realai import RealAIClient

# Use local models by default, fall back to API if needed
client = RealAIClient(
    api_key="sk-your-api-key",  # Optional fallback
    use_local=True               # Prefer local models
)

# This will use your local model
response = client.chat.create(messages=[...])

# Force API usage for a specific call
client._use_local = False
response = client.chat.create(messages=[...])  # Uses API
client._use_local = True  # Back to local
```

## 📊 Performance Tips

### Optimize Context Length

```python
# Register with smaller context for faster inference
python local_model_cli.py register my-model /path/to/model.gguf --context 1024
```

### GPU Layer Configuration

```python
# All layers on GPU (fastest)
python local_model_cli.py register my-model /path/to/model.gguf --gpu-layers -1

# Some layers on GPU (if you have limited VRAM)
python local_model_cli.py register my-model /path/to/model.gguf --gpu-layers 20

# CPU only
python local_model_cli.py register my-model /path/to/model.gguf --gpu-layers 0
```

## 🛠️ Troubleshooting

### Model Not Loading

1. Check if the model file exists:
   ```bash
   python local_model_cli.py list
   ```

2. Verify the model path is correct:
   ```bash
   python local_model_cli.py config
   ```

3. Try a smaller model first to test your setup

### Out of Memory

1. Use a quantized model (Q4_K_M or Q5_K_M)
2. Reduce context length: `--context 1024`
3. Offload fewer layers to GPU: `--gpu-layers 10`
4. Close other applications

### Slow Inference

1. Enable GPU acceleration (see GPU Acceleration section)
2. Use a smaller/quantized model
3. Reduce `max_tokens` in your requests
4. Ensure GPU drivers are up to date

## 📖 Python API Reference

### LocalModelManager

```python
from realai.local_models import get_model_manager

manager = get_model_manager()

# List models
models = manager.list_models()

# Register a model
from realai.local_models import LocalModelType
manager.register_model(
    "my-model",
    LocalModelType.LLM,
    {
        "path": "/path/to/model.gguf",
        "backend": "llama-cpp",
        "context_length": 2048,
        "gpu_layers": -1
    }
)

# Check if available
if manager.is_model_available("my-model"):
    print("Model ready!")
```

### LocalLLMEngine

```python
from realai.local_models import get_llm_engine

engine = get_llm_engine()

# Load a model
engine.load_model("llama3.2-1b")

# Generate text
response = engine.generate(
    "Explain AI in simple terms",
    max_tokens=200,
    temperature=0.7
)

# Chat completion
response = engine.chat_completion(
    messages=[
        {"role": "system", "content": "You are a helpful assistant"},
        {"role": "user", "content": "Hello!"}
    ],
    max_tokens=150
)
```
